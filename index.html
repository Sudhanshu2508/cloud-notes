Cloud Computing - Definition Cloud Computing is a model that enables on-demand access to a shared pool of configurable computing resources—such as networks, servers, storage, applications, and services—that can be rapidly provisioned and released with minimal management effort or service provider interaction. It offers scalable and flexible resources over the internet, allowing users to access and use IT services without owning physical infrastructure. Characteristics of Cloud Computing (As per NSIT - National Institute of Standards and Technology, NIST) NIST defines five essential characteristics of Cloud Computing: 1. On-Demand Self-Service o Users can automatically provision computing capabilities, like server time and storage, as needed, without requiring human interaction with service providers. 2. Broad Network Access o Services are accessible over the network and available through standard devices such as mobile phones, tablets, laptops, and desktops. 3. Resource Pooling o Cloud resources (storage, processing, memory, network bandwidth) are pooled to serve multiple users using a multi-tenant model, dynamically assigning and reassigning resources based on demand. 4. Rapid Elasticity o Resources can be scaled up or down quickly, sometimes automatically, depending on the demand. To the user, the resources often appear to be unlimited and available anytime. 5. Measured Service o Cloud systems automatically control and optimize resource usage through metering capabilities. This ensures transparency for both the provider and the user in terms of resource usage (pay-per-use model). Types of Cloud Deployment Models 1. Private Cloud A Private Cloud is a cloud environment dedicated to a single organization. It can be physically located on the company’s on-site data center or hosted by a third-party service provider, but the infrastructure is maintained on a private network. Key Features: • High level of security and privacy • Greater control over data and resources • Customized to specific organizational needs Examples: Government agencies, financial institutions, large enterprises with sensitive data 2. Public Cloud A Public Cloud is owned and operated by a third-party cloud service provider, which delivers computing resources like servers and storage over the Internet. Key Features: • Cost-effective (pay-as-you-go model) • Highly scalable • No need for infrastructure management by the user Examples of Providers: Amazon Web Services (AWS), Microsoft Azure, Google Cloud Platform (GCP) 3. Hybrid Cloud A Hybrid Cloud combines both private and public clouds, allowing data and applications to be shared between them. This provides greater flexibility, more deployment options, and helps optimize existing infrastructure, security, and compliance. Key Features: • Balanced approach: security + scalability • Supports data portability and interoperability • Useful for disaster recovery and burst workloads Use Case Example: An organization may use a public cloud for general computing but a private cloud to store sensitive data. Cloud Service Models 1. IaaS (Infrastructure as a Service) Definition: IaaS provides virtualized computing resources over the internet, such as servers, storage, and networking. Users can rent these resources and build their own IT platforms. Key Features: • Users manage: OS, applications, runtime • Providers manage: servers, storage, virtualization, networking Examples: • Amazon EC2 • Microsoft Azure Virtual Machines • Google Compute Engine Use Case: Startups hosting a website or app without buying physical servers. 2. PaaS (Platform as a Service) Definition: PaaS provides a ready-to-use development environment, where developers can build, test, and deploy applications without managing the underlying infrastructure. Key Features: • Users manage: applications and data • Providers manage: OS, runtime, servers, storage, networking Examples: • Google App Engine • Microsoft Azure App Services • Heroku Use Case: Developers building custom web or mobile apps quickly with built-in scalability. 3. SaaS (Software as a Service) Definition: SaaS delivers ready-to-use software applications over the internet. Users access them through web browsers or apps without installing or maintaining them. Key Features: • Everything managed by the provider • Access anytime, anywhere via the internet Examples: • Gmail • Microsoft Office 365 • Zoom • Dropbox Use Case: Using email, video conferencing, or office tools without local installation. 4. FaaS (Function as a Service) – also known as Serverless Definition: FaaS allows developers to run individual functions or pieces of code in response to events without managing servers or infrastructure. Key Features: • Event-driven execution • Scales automatically • Pay only when the function runs Examples: • AWS Lambda • Google Cloud Functions • Azure Functions Use Case: Trigger a function when a file is uploaded, or a user signs in—no need to run a full backend server. Cloud Service Management Cloud Service Management refers to the process of managing and delivering cloud services efficiently and effectively. It includes the tools and techniques used to monitor and control cloud resources, ensure service quality, meet SLAs, and optimize performance and cost. Key Aspects: • Service Level Agreement (SLA) Management • Incident and Problem Management • Change and Configuration Management • Usage and Billing Management • Capacity Planning Cloud Computing Economics Economics of cloud computing deals with cost efficiency, resource optimization, and value creation. Organizations need to understand both the direct costs and indirect costs associated with cloud adoption and operations. Key Concepts: 1. Capital Expenditure (CAPEX) vs Operational Expenditure (OPEX): • CAPEX refers to upfront investments in physical infrastructure (e.g., servers, storage). • OPEX represents ongoing costs for services used (e.g., cloud usage fees). 2. Total Cost of Ownership (TCO): • Includes all costs over a system's lifecycle, including acquisition, operations, and decommissioning. 3. Return on Investment (ROI): • The gain from investing in cloud computing relative to its cost. 4. Cost Models: • Pay-as-you-go: Charged based on actual usage. • Subscription: Fixed periodic fee. • Spot Pricing: Discounted pricing for spare capacity (e.g., AWS Spot Instances). Choosing a Cloud Provider Factors to consider: • Pricing and Billing Models • Compliance and Certifications (e.g., GDPR, HIPAA) • Data Center Locations • Security Measures and Encryption • Tools and APIs • Uptime and SLA Guarantees • Support and Documentation Major Cloud Providers: 1. Amazon Web Services (AWS): Leading provider with a wide range of services. 2. Microsoft Azure: Strong in hybrid cloud, integrates with Microsoft ecosystem. 3. Google Cloud Platform (GCP): Known for data analytics and AI tools. 4. IBM Cloud: Emphasizes enterprise and cognitive computing. 5. Oracle Cloud: Database-centric services. Managing Cloud Infrastructure Includes provisioning, monitoring, and maintaining cloud resources: • Auto-scaling: Automatically adjusts compute resources based on load. • Load Balancing: Distributes traffic across multiple instances. • Monitoring Tools: CloudWatch (AWS), Stackdriver (GCP), Azure Monitor. • Configuration Management: Tools like Ansible, Puppet, Chef. Security, Availability, and Reliability These three are critical to cloud service delivery: • Security: Ensuring data confidentiality, integrity, and availability. • Availability: Services should be accessible when needed. • Reliability: Consistent performance and quick recovery from failure. Cloud Governance Governance ensures the effective use of cloud resources aligned with business goals. It includes: • Policy Enforcement • Access Controls • Resource Tagging • Budgeting and Usage Tracking Cost Optimization Techniques • Rightsizing Instances • Using Reserved or Spot Instances • Scheduling on/off times for non-critical workloads • Leveraging Free Tiers and Open Source Tools • Continuous Cost Monitoring This module emphasizes understanding the operational and financial considerations of using cloud platforms effectively.

Advantages: - Cloud-native and scalable - Extensive ecosystem - Strong community and support - Frequent updates and innovations Salesforce provides a comprehensive suite of cloud services that cater to a wide range of business functions, making it a central player in the cloud CRM space.
Application Development in Cloud Computing Application development in cloud computing involves building and deploying applications that utilize cloud infrastructure, platforms, and services. It enhances scalability, flexibility, and speed of development. Key Concepts: 1. Cloud-native Applications: - Designed specifically to run in the cloud environment. - Built using microservices architecture, containers, CI/CD pipelines, and scalable storage. 2. Cloud Application Environments: • Amazon Web Services (AWS): - Elastic Beanstalk: Easy-to-use service for deploying apps. - AWS Lambda: Serverless computing to run code in response to events. - Cloud9: Cloud-based integrated development environment (IDE). - EC2: Virtual servers for deploying and scaling applications. • Microsoft Azure: - App Services: Host web apps and APIs. - Azure Functions: Serverless platform for event-driven development. - Visual Studio Code Spaces: Collaborative cloud IDE. - Azure Kubernetes Service (AKS): Deploy containerized applications. • Google Cloud Platform (GCP): - App Engine: Platform for deploying web apps. - Cloud Functions: Lightweight serverless execution. - Cloud Run: Fully managed container execution environment. - Firebase: Mobile and web app development backend. Development Life Cycle: - Planning and Requirements Analysis - Designing the Architecture (Monolith vs Microservices) - Coding (language and tools suited for cloud) - Testing (automated and manual testing) - Deployment (using CI/CD tools) - Monitoring (real-time logging and error detection) Tools Used in Cloud Development: - GitHub, GitLab (source control) - Jenkins, CircleCI (CI/CD automation) - Docker (containerization) - Kubernetes (orchestration) - Terraform (infrastructure as code) - Postman (API testing) Benefits of Cloud-based App Development: - Faster time to market - Reduced infrastructure cost - Scalable architecture - Global accessibility Security Considerations: - Secure authentication & authorization (OAuth, JWT) - Data encryption (in transit & at rest) - Regular vulnerability scanning - Role-based access control (RBAC) Summary: Cloud platforms simplify application development by offering scalable environments, built-in services, automation tools, and global infrastructure, empowering developers to focus more on features and user experience than server management.
Cloud Security & Risk Management Cloud security is a set of policies, technologies, and controls that protect cloud data, applications, and infrastructure. It ensures confidentiality, integrity, availability, and compliance of cloud-based resources. Key Security Risks: 1. Data Breaches: - Unauthorized access or exposure of sensitive data. - Causes: weak passwords, misconfigured storage, poor encryption. 2. Insecure APIs: - APIs are entry points to services. If poorly secured, they can be exploited. 3. Denial of Service (DoS/DDoS): - Overwhelming a cloud service to make it unavailable. 4. Insider Threats: - Employees or contractors abusing access privileges. 5. Misconfigured Cloud Settings: - Publicly accessible S3 buckets, improper IAM roles. Identity and Access Management (IAM): - IAM controls who has access to what in the cloud environment. - Includes role-based access control (RBAC), multi-factor authentication (MFA), least privilege principle. Compliance & Regulations: - Cloud providers often comply with standards like: • GDPR (General Data Protection Regulation) • HIPAA (Health Insurance Portability and Accountability Act) • ISO/IEC 27001 • SOC 2 Cloud Security Best Practices: • Encrypt sensitive data (in transit and at rest) • Regular vulnerability assessments • Use firewalls and network segmentation • Enable logging and monitoring (e.g., AWS CloudTrail, Azure Security Center) • Backup critical data regularly • Train employees on security awareness Governance, Risk, and Compliance (GRC): - Governance: Policies and frameworks to guide secure operations - Risk: Identify, assess, and mitigate threats - Compliance: Adhere to regulatory and contractual requirements Security Tools in Cloud Platforms: - AWS: GuardDuty, Macie, Inspector, Security Hub - Azure: Security Center, Sentinel, Defender for Cloud - GCP: Security Command Center, VPC Service Controls Cloud Adoption Strategy (Viksit Bharat @2047): - Government of India initiative to modernize digital infrastructure by 2047. - Phased adoption for public sector and education. - Emphasis on secure, inclusive, and scalable cloud solutions. Conclusion: A robust cloud security posture requires shared responsibility between providers and users, continuous monitoring, and a culture of security awareness.
